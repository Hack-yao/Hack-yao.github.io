6.1 创建对象

6.1.1 对象直接量

6.1.2 通过new创建对象

new 运算符创建并初始化一个新对象。关键字new后面紧跟着一个函数调用。该函数称为构造函数(constructor),构造函数用来初始化
新创建的对象。

6.1.3 原型

每一个javascript对象(除了null)都和另外一个对象关联。“另一个”对象就是我们所知的原型，每一个对象都从原型继承属性。

  • 所有通过对象直接量创建的对象都有相同的一个原型对象，即为Object.prototype.
  • 通过new 构造函数() 创建的对象的 原型就是 构造函数的prototype的属性值。
  • new Object() 产生的对象的原型 也是 Object.prototype

6.1.4 Object.create()

Object.create(obj) 创建一个新对象，其中第一个参数obj就是这个对象的原型。第二个可选参数，用以对对象的属性进行进一步描述。

  • Object.create(null) 创建一个没有原型的新对象，但通过这种方式创建的对象不会继承任何东西，甚至不会包括基础方法，比如：toString(),
    也就是说不能和 + 运算符一起正常工作：

  • Object.create(Object.prototype) c创建一个普通的空对象。和({ } 或 new Object() 创建的对象)

模拟Object.create 方法—不支持通过传入null原型来创建对象

inherit()函数的其中一个用途就是防止库函数无意 修改那些不受你控制的对象。 不是将对象直接作为参数传入函数，而是将它的 继承对象 传入函数。当函数
读取该继承对象的属性时，实际上读取的是继承来的值。如果给继承对象赋值,则这些属性只会影响这个继承对象自身，而不是原始对象。

6.2 属性的查询和设置

  • 通过(.)运算符来获取属性的值

    点(.)右侧必须是一个以属性名称命名的简单标识符，不能是变量和字符串

  • 通过[]运算符来获取属性的值

    []内必须是一个计算结果为字符串的表达式或者可以转换为字符串的值

  • (.) 与 [] 的差异

    点运算符后的标识符不能是保留字。 o.for or o.class 是非法的。如果一个对象的属性是保留字
    则只能用[]的形式来访问，比如 o[‘for’]和o[‘class’].

    ECMAScript5 放松了此限制。可以在(.)运算符后面直接使用保留字

6.2.2 继承

Javascript对象具有“自有属性”（own property), 也有一些属性是从原型对象继承而来的。
查询对象的属性时，会沿着原型链查找。

属性赋值操作首先检查原型链，以此判定是否允许赋值操作。例如，如果o继承自一个只读属性x,那么赋值操作是不允许的。如果允许赋值操作
他也是在原始对象上创建属性或者对已有的属性赋值，而不会去修改原型链。在Javascript中，只有查询属性时才会体会到继承的存在，而设置属性则和继承无关。
这个重要特性可以让程序员可以有选择的 覆盖 继承的属性。

属性赋值要么失败，要么创建一个属性，要么在原始对象中设置属性。但有一个例外：
如果o继承自属性x，而这个属性是一个具有setter方法的accessor属性。那么这时将调用setter方法而不是给o创建一个属性x。
setter 方法是由对象o调用的，而不是定义这个属性的原型对象调用的。 因此如果setter方法定义任意属性，
这个操作只是针对o本身，并不会修改原型链。

6.3 删除属性

delete运算符可以删除对象的属性。操作数是一个 属性访问表达式。

delete 只能删除自有属性，不能删除继承属性–要删除继承属性必须从定义这个属性的原型对象上删除，但这会影响所以继承这个原型的对象。

delete只是断开属性和宿主对象的联系，而不会去操作属性中的属性

delete 不能删除那些可配置性为false的属性（尽管可以删除不可扩展对象的可配置属性。）。某些内置对象的属性是不可配置的

通过变量声明和函数声明创建的全局对象的属性。严格模式下删除不可配置属性会报类型错误。非严格返回false

delete 的这种属性可能造成内存泄漏，所以在销毁对象的时候，要遍历属性中的属性，依次删除。

delete的返回值

  • delete表达式删除成功或者没有任何副作用（比如删除不存在的属性）时，返回true. 如果delete后面不是
    一个 属性访问 表达式，delete同样返回true。
  • delete 在 严格模式下 删除不可配置属性会报类型错误，非严格 返回false

通过变量声明和函数声明创建的全局对象的属性,是不可配置属性不能删除

  • 严格模式下删除，需要制定对象以及属性

6.4 属性检测

Javascript对象可以看做属性的集合。想要判断某个属性是个存在于某个对象中。可以通过以下方法：
1.in 运算符
‘属性名’ in 对象： 如果对象的自有属性或继承属性中包含 这个属性名 则返回true

2.hasOwnProperty() 检测给的的名字是否是对象的自有属性。对于继承属性返回false

3.propertyIsEnumerable() 检测是否是自有属性，且可以被枚举。 enumerable attribute

6.5 枚举属性

  • for/in
  • Object.keys(obj) – 返回对象中 可枚举的 自有属性名称 组成的 数组
  • Object.getOwnPropertyNames() — 返回对象的 所有自有属性 的名称组成的 数组

6.6 属性getter和setter

对象属性是由名字、值和一组特性构成的。在es5中属性值可以用一个或两个方法替代-getter和setter（非常类似Ruby中的）
。用setter和getter定义的属性就是 accessor 属性。不同于数据属性，数据属性只有一个简单的值。

存取属性的 读/写性
同时拥有getter和setter–读/写属性
getter — 只读属性
setter— 只写属性

6.7 属性的特性

  • 数据属性

    • value 值
    • 可写性 writable
    • 可枚举性 enumerable
    • 可配置性 configurable

  • 存储器属性

    • 读 get
    • 写 set
    • 可枚举 enumerable
    • 可配置性 configurable

  • 查询属性的特性
    Object.getOwnPropertyDescriptor(obj, “属性名”); 可以获取某个对象特定 自有属性的 属性描述符

  • 设置属性的特性

    Object.defineProperty(obj, prop, descriptor) 方法会直接在一个对象上定义一个新属性，或者修改一个已经存在的属性， 并返回这个对象。

  • 修改多个属性
    Object.defineProperties() 方法在一个对象上添加或修改一个或者多个自有属性，并返回该对象。

    • 语法

      Object.defineProperties(obj, props)

6.8 对象的三个属性

每个对象都有与之相关的 原型prototype，类 class， 和可扩展性extensible attribute。

6.8.1 原型属性 The prototype Attribute

6.8.2 The class Attribute

类属性是一个字符串，用于对象的类型信息–Es5中通过Object.prototype.toString()返回对象的类信息
很多对象的toString()方法重写了。为了调用正确的toString()版本，必须间接的调用Function.call()方法。

classof()函数可以实现返回任意对象的类

6.8.3 可扩展性

对象的可扩展性用以表示是否可以给对象添加新属性。
所有的内置对象和自定义对象都是可扩展的，宿主对象的可扩展性是由Javascript引擎所决定的。
